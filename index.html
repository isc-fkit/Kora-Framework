II · Luồng phân tích yêu cầu (đầu ra chuẩn)

Mọi phân tích đi qua 4 cổng tuân thủ — ĐÚNG THỨ TỰ, đảm bảo đầu ra chuẩn · nhất quán · chuyên nghiệp · trace được nguồn:

Tuân thủ RULE DOMAIN

Đọc config/domain-rules.md (theo preset đã chọn — Healthcare / Fintech / E-commerce / Retail…) trước mỗi phân tích; áp thuật ngữ, ngưỡng và rule thận trọng của ngành (vd y tế: phân biệt "hiển thị thông tin" vs "tư vấn y tế", không chẩn đoán thay bác sĩ).

Theo WORKFLOW phân tích

Bám workflows/03-request.md + quy trình BA 8 bước (B0→B7): đọc KB (index + vault + lessons) → phát hiện xung đột · tác động · lỗ hổng → trình bày tiếng Việt trích nguồn theo file → ✋ Approval Gate trước khi ghi.

Áp TEMPLATE PROMPT (theo vai trò)

Theo vai trò được hỏi mỗi khi có yêu cầu/tính năng mới (BA / PO / SA / QA) nạp templates/prompts/role-<x>.md + thư viện ba-prompt-library.md → mỗi artifact (US / AC / BR / FR / NFR / validation / test…) ra đúng ĐỊNH DẠNG CHUẨN TỰ ĐỘNG, user không cần yêu cầu "viết theo format X".
